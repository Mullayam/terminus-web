1. OVERVIEW
===================================================

The monaco-editor module was reorganized from a flat lib/ directory into
categorized subdirectories. All imports across the codebase have been updated.
The barrel export (lib/index.ts) and module root (index.ts) remain the primary
public API — consumers don't need to know about internal subdirectory paths.

3. CONTEXT ENGINE ARCHITECTURE
===================================================

The context engine has a SHARED layer and a MONACO-SPECIFIC layer:

  Shared (src/lib/context-engine/):
    - contextEngineApi.ts    — CDN fetch, manifest loading, version tracking
    - contextEngineStorage.ts — IndexedDB persistence (languages + commands)
    - index.ts               — Barrel re-export

    Consumers:
    ✓ Monaco: ContextEnginePanel, contextEngineProviders, MonacoEditor
    ✓ SSH:    Terminal.tsx, rightSidebar.tsx, commandPacks.tsx
    ✓ Editor: EditorTerminalWithGhost.tsx

  Monaco-specific (src/modules/monaco-editor/lib/context-engine/):
    - contextEngineConverters.ts — IndexedDB data → Monaco provider adapters
    - contextEngineProviders.ts  — Registration orchestrator (25 provider types)
    - index.ts                   — Barrel

  Decision: Shared layer stays in src/lib/ because SSH terminal and
  editor terminal also consume it. Monaco-side stays local because it's
  tightly coupled to monaco.languages API.


===================================================
4. FILE SYSTEM PROVIDER ARCHITECTURE (lib/filesystem/)
===================================================

  Interfaces:
    FileSystemProvider extends FileOperationHandlers
    - connect/disconnect/status/readdir (connection lifecycle)
    - readFile/writeFile/createFile/createDir/rename/move/copy/delete (operations)

  Composable handlers:
    createSftpHandlers(socket)  — socket.io operations
    createApiHandlers(apiUrl)   — REST API operations
    composeHandlers(...sources) — mix-and-match per operation

  Performance features:
    - DirCache: LRU + TTL in-memory directory cache
    - filterEntries: Smart ignore (node_modules, .git, dist, etc.)
    - paginateEntries: Cursor + offset pagination with nextCursor
    - useFileSystemTree: Cache-aware hook with stale-while-revalidate

  UI (components/EditorFileTree.tsx):
    - Virtualized via react-window v2 (List component)
    - VS Code toolbar: New File, New Folder, Refresh, Collapse All, Toggle Hidden
    - Auto-loading infinite scroll on near-bottom detection
    - Dot files shown by default (showHiddenFiles = true)

5. IMPORT PATH MIGRATION
===================================================

All consumer imports have been updated from flat lib/ paths to categorized
subdirectory paths. Examples:

  BEFORE                                          AFTER
  ──────                                          ─────
  @/modules/monaco-editor/lib/language-groups      @/modules/monaco-editor/lib/language/language-groups
  @/modules/monaco-editor/lib/file-system-types    @/modules/monaco-editor/lib/filesystem/file-system-types
  @/modules/monaco-editor/lib/monacoThemeToXterm   @/modules/monaco-editor/lib/themes/monacoThemeToXterm
  ../lib/extensionLoader                           ../lib/extensions/extensionLoader
  ../lib/hoverProvider                             ../lib/hover/hoverProvider
  ../lib/contextEngineProviders                    ../lib/context-engine/contextEngineProviders
  ./remote-providers                               ./providers

The barrel (lib/index.ts) re-exports everything from the new paths,
so consumers importing from "@/modules/monaco-editor" or "./lib" are unaffected.
