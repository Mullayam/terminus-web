Integration Points:

    for content access, lifecycle tied to editor instance.

    Move formatting off the main thread. Currently blocks UI on large files.

  - src/modules/monaco-editor/lib/remote-providers/index.ts
    Offload completion/hover lookups for 94 languages to a worker with mirror models.

  - src/modules/editor/workers/lint.worker.ts
    Port structural linter from raw postMessage to Monaco proxy.

=================================================================

Registrations (4 total):
  A. LSP Bridge
     File: src/modules/monaco-editor/lib/lsp/providers.ts (line 208)
     Delegates to client.documentSymbol()

  B. Remote-Providers (CDN/JSON)
     File: src/modules/monaco-editor/lib/remote-providers/adapters.ts (line 425)
     Regex-per-line pattern matching from JSON

  C. Context Engine (IndexedDB)
     File: src/modules/monaco-editor/lib/contextEngineProviders.ts (line 161)
     Runs patterns with gm flag on full model text

  D. Plugin System
     File: src/modules/monaco-editor/core/plugin-context.ts (lines 234-239)

  All are single-file (by API design: provideDocumentSymbols(model))

Symbol Pattern Storage (2 layers, NOT shared with DocumentSymbolProvider):
  1. Static patterns: src/modules/monaco-editor/lib/symbol-patterns.ts (lines 35-488)
     SYMBOL_PATTERNS array, ~25 language families, lazy Map index via getLangIndex()
     Used by: CodeLens (buildCodeLenses), AI context extraction (findSymbolLines)

  2. CDN-loaded dynamic patterns: symbol-patterns.ts (lines 561-619)
     Fetches @enjoys/context-engine documentSymbol/{langId}.json
     Cached in CacheStorage (terminus-doc-symbols-v1)
     Injected into getLangIndex() map

  NOTE: CodeLens/AI patterns are SEPARATE from DocumentSymbolProvider registrations.
  They are not shared — two different systems scanning symbols independently.

Breadcrumb/Outline UI:
  - breadcrumbs.ts (src/modules/editor/plugins/builtin/breadcrumbs.ts)
    Own scope detection via brace/indent tracking, INDEPENDENT of DocumentSymbolProvider
  - Monaco's built-in Outline/breadcrumbs are driven by DocumentSymbolProvider automatically
  - PathBreadcrumb.tsx (src/pages/sftp/components/) — SFTP file path only, not code symbols

Gaps:
  - Breadcrumbs plugin duplicates work (own scope detection vs DocumentSymbolProvider)
  - No symbol caching across edits (re-runs all regex every call)
  - Remote-providers (line-by-line) and context-engine (full-text gm) use inconsistent approaches

=================================================================
Supporting Infrastructure
=================================================================

Symbol Storage / Caching:
  - Static pattern index (symbol-patterns.ts): Lazy Map, per-language RegExp[]
  - CacheStorage (CDN cache): terminus-doc-symbols-v1, caches JSON for offline
  - IndexedDB (context-engine): terminus-context-languages Dexie DB, per provider type
  - NO runtime symbol index aggregating symbols found across open files

File Content Access:
  - SFTP fetch: src/lib/api.ts ApiCore.fetchFileContent(sessionId, remotePath)
  - Monaco models: model.getValue() for focused editor (in-memory only)
  - GitHub: src/modules/monaco-editor/extensions/githubApi.ts fetchFileContent()

editorSftpStore:
  - src/store/editorSftpStore.ts tracks CONNECTION STATE ONLY (host, username, status)
  - Does NOT track open files, file contents, or file paths
  - Open tabs tracked in ViewPanelStore (src/modules/monaco-editor/view-panel/store.ts)
    as openTabs: ViewPanelTab[] — tab metadata only (tabId, panelId, props), not file content
